# 開発ガイドライン (developmet Guidelines)

開発お疲れ様です。
開発をスムーズに進めるため、以下のルールに従って作業をお願いします。

## 1. ブランチ戦略 (Branching Strategy)

* **ベースブランチ:** 原則として `main` ブランチから新しいブランチを作成してください。
* **ブランチ名の命名規則:** 作業内容に応じて以下のプレフィックスを使用してください。
    * 新機能: `feature/機能名` (例: `feature/add-login`)
    * バグ修正: `fix/内容` (例: `fix/header-layout`)
    * リファクタリング: `refactor/内容`
    * その他: `chore/内容`

## 2. 開発プロセスとチェック項目 (Development Process)

プルリクエスト（PR）を作成する前に、**必ず以下の3つのチェックをローカル環境で通過させてください。**

これらのコマンドが失敗する状態でのプルリクエストはマージされません。

1. **Type Check (型チェック)**
   ```bash
   # エラーが出ないことを確認してください
   npmp typecheck
   ```

2. **Lint (静的解析)**
   ```bash
   # コードスタイルの修正を行ってください
   pnpm lint
   ```

3. **Test (テスト)**
   ```bash
   # 全てのテストがパスすることを確認してください
   pnpm test
   ```

## 3. プルリクエスト (Pull Request)

* PRのタイトルは変更内容がわかるように記述してください。
* レビューを依頼する前に、上記の `typecheck`, `lint`, `test` が通っていることを再確認してください。
